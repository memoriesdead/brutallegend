# Contributing to Brutal Legend Reverse Engineering

## Welcome

Thank you for your interest in contributing. This project needs reverse engineers, developers, and documenters.

## Ways to Contribute

### Reverse Engineering
- Analyze BrutalLegend.exe in Ghidra
- Trace function calls in x64dbg
- Document discovered functions

### Tool Development
- Build dfpf-toolkit (C++)
- Create Python automation scripts
- Develop mod loader framework

### Documentation
- Write format specifications
- Create tutorials
- Document discoveries

## Getting Started

1. Read [docs/tutorials/01_GETTING_STARTED.md](docs/tutorials/01_GETTING_STARTED.md)
2. Pick a TODO item
3. Make your changes
4. Submit a pull request

## Standards

### Code
- C++: Follow ISO standard, use modern features
- Python: PEP 8 style
- Document all functions

### Documentation
- Use clear, concise language
- Include examples where possible
- Update TODO status when complete

### Commits
- Use conventional commits: `feat:`, `docs:`, `fix:`, `refactor:`
- Reference issues where applicable
